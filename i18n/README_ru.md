# Платформа UGJB

[![GitHub](https://img.shields.io/badge/github-repo-blue?logo=github)](https://github.com/qtvhao/UGJB)
[![License](https://img.shields.io/github/license/qtvhao/UGJB.svg?color=blue)](LICENSE)

**Единая платформа для управления персоналом и инженерной аналитики** - Модульная платформа, объединяющая управление персоналом и анализ производительности инженерных команд в единой системе с открытым исходным кодом.

## Обзор

Платформа UGJB позволяет организациям согласовывать кадровые решения с техническими результатами через инсайты в реальном времени. Она устраняет фрагментированные SaaS-решения, объединяя возможности корпоративного уровня со стандартизированными паттернами интеграции и переиспользуемыми компонентами, при этом снижая совокупную стоимость владения.

### Основные возможности

- **Единое управление персоналом** - Профили сотрудников, отслеживание навыков, распределение FTE и статус работы
- **Инженерная аналитика** - Метрики DORA, показатели качества кода и индикаторы надежности
- **Интеграция с инструментами разработки** - Jira, GitLab, Firebase Crashlytics, Prometheus
- **Дашборды в реальном времени** - Настраиваемая визуализация KPI и отчеты
- **Контроль доступа на основе ролей** - Детальные разрешения и безопасность данных
- **Открытый исходный код и модульность** - Расширяемая архитектура без привязки к вендору

## Быстрый старт

### Предварительные требования

- Docker и Docker Compose
- Git

### Установка

```bash
# Клонировать репозиторий
git clone https://github.com/qtvhao/UGJB.git
cd UGJB

# Запустить все сервисы
docker-compose up -d

# Проверить endpoint здоровья
curl http://localhost:8080/health
```

### Доступ к платформе

- **Web UI**: http://localhost:8081
- **API Gateway**: http://localhost:8080

## Архитектура

UGJB следует микросервисной архитектуре с четко определенными ограниченными контекстами:

- **Управление персоналом** - Реестр сотрудников и движок распределения
- **Инженерная аналитика** - Сборщик метрик, движок KPI, дашборд инсайтов
- **Управление целями** - Отслеживание целей и ключевых результатов
- **Управление проектами** - Координация спринтов и распределение задач
- **Системная интеграция** - Пайплайн данных и API-шлюз
- **Благополучие персонала** - Прогнозирование выгорания и мониторинг благополучия

## Почему UGJB?

### Решаемые проблемы

1. **Фрагментация интеграций** - Объединяет данные из Firebase, Prometheus, GitLab и Jira
2. **Изолированные домены** - Связывает управление навыками HR с инженерными KPI
3. **Барьеры стоимости** - ≤ $120k TCO за 3 года против $200k+ корпоративного SaaS
4. **Ограничения кастомизации** - Расширяемые рабочие процессы с сохранением стабильности платформы

### Метрики успеха

| Метрика | Базовая линия | Цель |
|---------|--------------|------|
| TCO за 3 года | $201k-$246k | ≤ $120k |
| Покрытие интеграций | 50% GitLab | 100% покрытие |
| Время до инсайтов | 72+ часа | ≤ 2 часа |
| Uptime платформы | Не определено | ≥ 99.9% |

## Базовое использование

### Управление сотрудниками

```bash
# Создать профиль сотрудника через API
curl -X POST http://localhost:8080/api/v1/employees \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Иван Иванов",
    "role": "Старший разработчик",
    "department": "Инженерия",
    "status": "active",
    "fte": 100
  }'
```

### Просмотр инженерных метрик

Доступ к дашборду инженерных метрик:
- Метрики DORA (частота развертываний, время выполнения)
- Показатели качества кода
- Недавние развертывания
- Производительность команды

### Настройка интеграций

Подключение внешних инструментов через Web UI:
1. Перейти в «Интеграции»
2. Выбрать тип инструмента (Jira, GitLab, Firebase, Prometheus)
3. Ввести API endpoint и аутентификацию
4. Установить частоту синхронизации

# منصة UGJB

[![GitHub](https://img.shields.io/badge/github-repo-blue?logo=github)](https://github.com/qtvhao/UGJB)
[![License](https://img.shields.io/github/license/qtvhao/UGJB.svg?color=blue)](LICENSE)

**منصة موحدة لإدارة القوى العاملة وتحليلات الهندسة** - منصة نمطية تدمج إدارة الموارد البشرية وتحليل أداء الهندسة في نظام مفتوح المصدر واحد.

## نظرة عامة

تمكن منصة UGJB المؤسسات من مواءمة قرارات المواهب مع النتائج التقنية من خلال رؤى في الوقت الفعلي. إنها تزيل حلول SaaS المجزأة من خلال الجمع بين قدرات مستوى المؤسسات مع أنماط التكامل الموحدة والمكونات القابلة لإعادة الاستخدام، مع تقليل التكلفة الإجمالية للملكية.

### الميزات الرئيسية

- **إدارة موحدة للقوى العاملة** - ملفات تعريف الموظفين، تتبع المهارات، تخصيص FTE وحالة العمل
- **تحليلات الهندسة** - مقاييس DORA، درجات جودة الكود ومؤشرات الموثوقية
- **التكامل مع أدوات التطوير** - Jira، GitLab، Firebase Crashlytics، Prometheus
- **لوحات معلومات في الوقت الفعلي** - تصورات KPI وتقارير قابلة للتخصيص
- **التحكم في الوصول على أساس الأدوار** - أذونات دقيقة وأمان البيانات
- **مفتوح المصدر ونمطي** - بنية قابلة للتوسيع، بدون ارتباط بموردين

## البدء السريع

### المتطلبات الأساسية

- Docker و Docker Compose
- Git

### التثبيت

```bash
# استنساخ المستودع
git clone https://github.com/qtvhao/UGJB.git
cd UGJB

# بدء جميع الخدمات
docker-compose up -d

# التحقق من نقطة نهاية الصحة
curl http://localhost:8080/health
```

### الوصول إلى المنصة

- **واجهة الويب**: http://localhost:8081
- **بوابة API**: http://localhost:8080

## البنية المعمارية

تتبع UGJB بنية معمارية قائمة على الخدمات الصغيرة مع سياقات محددة بوضوح:

- **إدارة الموارد البشرية** - سجل الموظفين ومحرك التخصيص
- **تحليلات الهندسة** - جامع المقاييس، محرك KPI، لوحة معلومات الرؤى
- **إدارة الأهداف** - تتبع الأهداف والنتائج الرئيسية
- **إدارة المشاريع** - تنسيق السبرينت وتعيين المهام
- **تكامل النظام** - خط أنابيب البيانات وبوابة API
- **رفاهية القوى العاملة** - التنبؤ بالإرهاق ومراقبة الرفاهية

## لماذا UGJB؟

### المشاكل المحلولة

1. **تجزئة التكامل** - توحيد البيانات من Firebase و Prometheus و GitLab و Jira
2. **صوامع النطاق** - ربط إدارة مهارات الموارد البشرية بمؤشرات KPI الهندسية
3. **حواجز التكلفة** - ≤ 120 ألف دولار TCO لمدة 3 سنوات مقابل 200 ألف دولار + SaaS للمؤسسات
4. **قيود التخصيص** - سير عمل قابل للتوسيع يحافظ على استقرار المنصة

### مقاييس النجاح

| المقياس | الخط الأساسي | الهدف |
|---------|-------------|--------|
| TCO لمدة 3 سنوات | 201 ألف - 246 ألف دولار | ≤ 120 ألف دولار |
| تغطية التكامل | 50% GitLab | 100% تغطية |
| الوقت حتى الرؤى | 72+ ساعة | ≤ 2 ساعة |
| وقت تشغيل المنصة | غير محدد | ≥ 99.9% |

## الاستخدام الأساسي

### إدارة الموظفين

```bash
# إنشاء ملف تعريف موظف عبر API
curl -X POST http://localhost:8080/api/v1/employees \
  -H "Content-Type: application/json" \
  -d '{
    "name": "أحمد محمد",
    "role": "مطور أول",
    "department": "الهندسة",
    "status": "active",
    "fte": 100
  }'
```

### عرض مقاييس الهندسة

الوصول إلى لوحة معلومات مقاييس الهندسة:
- مقاييس DORA (تكرار النشر، وقت التسليم)
- درجات جودة الكود
- عمليات النشر الأخيرة
- إنتاج الفريق الهندسي

### تكوين التكاملات

توصيل الأدوات الخارجية عبر واجهة الويب:
1. الانتقال إلى "التكاملات"
2. تحديد نوع الأداة (Jira، GitLab، Firebase، Prometheus)
3. إدخال نقطة نهاية API والمصادقة
4. تعيين تكرار المزامنة
